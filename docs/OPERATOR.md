# NameForge — запуск для оператора

Desktop-приложение: **Streamlit** + локальная **SQLite** + API **МойСклад**. Подробные правила домена — [`PROJECT_CONTEXT.md`](../PROJECT_CONTEXT.md).

## 1. Подготовка

1. Установите зависимости из корня репозитория (где лежит `requirements.txt` / окружение проекта).
2. Скопируйте `.env.example` в `.env` и задайте:
   - `MS_API_TOKEN` (или `MS_TOKEN` / `MOYSKLAD_TOKEN`) — токен API МойСклад; альтернатива — пара `MS_LOGIN` + `MS_PASSWORD`;
   - при необходимости `DB_PATH` — путь к SQLite (по умолчанию `data/autoname.db`);
   - `DRY_RUN` — по умолчанию **true** (безопасный режим без PATCH). Значение `false` допускается только когда нужна реальная запись в МойСклад; при старте приложения выводится предупреждение. В UI остаётся переключатель DRY RUN.
3. Убедитесь, что `config/attr_map.json` соответствует вашему аккаунту (генерация и обновление — `scripts/setup_ms_attributes.py`).

## 2. Запуск приложения

```bash
python -m streamlit run app.py
```

В боковой панели: статистика по статусам, переключатель **DRY RUN**, кнопка обновления справочника атрибутов (после смены метаданных в МойСклад).

## 3. Типовой рабочий цикл

1. **Импорт** — загрузка каталога из МойСклад в SQLite: `python scripts/import_from_ms.py` (см. раздел 4). При необходимости отчёт по заполненности полей: `python scripts/audit_import.py` → `docs/IMPORT_AUDIT.md`.
2. **Очередь** — фильтры по статусу и типу применимости, поиск, открытие карточки; вкладка **Ошибки** для статуса `error` и сброса в `new`.
3. **Карточка** — правка полей и применимости (для `fitment`), **Сгенерировать preview**, кнопка **Обновить из МС** (подтянуть атрибуты в локальную БД, кроме заблокированных по имени), при необходимости **Подтвердить и отправить** (если DRY RUN выкл.).
4. **Синхронизация** — предпросмотр одного товара, пакетная генерация preview и отправка review.
5. **Наименования** — разбор и сборка канонического формата имени (`src/naming/`), валидация и CSV.
6. **Шаблоны** — версии шаблонов имени в SQLite (активная версия + `applicability_type`).
7. **Алиасы** — нормализация «грязных» строк к каноническим значениям перед генерацией.

### Как добавить новый шаблон

В **Шаблоны** создайте строку: уникальная пара `template_key` + `version`, поле `applicability_type` (`fitment` или `universal`), шаблон строки `name_pattern`, флаг «активен» только у одной версии на пару ключ + тип (как принято в вашем процессе). Товары ссылаются на шаблон через `template_key` / `template_version` в SQLite.

### Алиасы для нормализации

В **Алиасы** задайте тип (`brand`, `part_type`, `make`, …), необязательный `scope_value`, исходную строку и каноническое значение. Генератор подставляет канон при совпадении нормализованного алиаса (см. `PROJECT_CONTEXT` по `ALIAS_TYPES`).

### Устранение типовых проблем

- **Пустой бренд / тип детали** — проверьте атрибуты в МойСклад и импорт; при имени в формате `{{категория}} … [[бренд]]` используйте страницу **Наименования** и повторный импорт (подстановка из имени при импорте).
- **Ошибки PATCH** — убедитесь, что `DRY_RUN` в `.env` и переключатель в UI отражают намерение; проверьте токен и лимиты API.
- **Заблокированное имя** — в карточке и боковой панели доступна разблокировка; синхронизация имени для `name_locked=1` не выполняется.

## 4. Импорт каталога из МойСклад

Тестовый прогон (первые 20 позиций):

```bash
python scripts/import_from_ms.py --limit 20
```

Полный импорт — без `--limit` (см. справку скрипта).

Сравнение хэша источника с текущими данными в МойСклад **без записи в БД**:

```bash
python scripts/import_from_ms.py --check
```

## 5. Безопасность

Не коммитьте `.env` с секретами. В проде МойСклад не дергайте скриптами без явного разрешения на задачу.
