# Архитектура "Золотого каталога" и Полная Матрица шаблонов NameForge 2.0

> Дорожная карта фаз проекта: [`DEVELOPMENT_PLAN_V2.md`](DEVELOPMENT_PLAN_V2.md).

Этот документ описывает строгие правила генерации наименований товаров для интеграции Odoo 19 с системой `AutoPrice_Matcher_v2`.

## 1. Синергия с AutoPrice_Matcher_v2 (ЧТО ДЕЛАТЬ ЗАПРЕЩЕНО)
NameForge 2.0 генерирует товары для Odoo 19, которые затем обрабатывает система `AutoPrice_Matcher_v2` (использующая FAISS/SBERT, нечеткий поиск и свои словари).
Следовательно, в коде `template_service.py` **ЗАПРЕЩЕНО**:
1. **Добавлять Артикул (SKU) в итоговое наименование.** Мэтчер сам тянет его из выделенного поля (fatt_str_8).
2. **Делать транслитерацию брендов.** Мэтчер сам нормализует бренды через свои обучаемые словари.

## 2. Три Золотых Правила Генерации (Core Rules для template_service.py)
При реализации логики склейки (конкатенации атрибутов по шаблону) обязательно реализуйте следующие пайплайны очистки:

**Правило 1: Семантическая плотность для FAISS (Удаление "Универсальный")**
Жестко вырезайте слова "Универсальный/Универсальная" из атрибута `Применяемость` для запчастей, ламп, расходников и химии. Если 30% каталога содержит это слово, семантический поиск FAISS в Мэтчере размывается. 
*Исключение:* Оставляем это слово ТОЛЬКО для аксессуаров (коврики, чехлы, накидки, щетки).

**Правило 2: Разделение Золотого имени и Поискового вектора (Для Odoo 19)**
В базовое `Product Name` (которое пойдет в печать и FAISS) синонимы (ПТФ, Туманки) добавлять нельзя! Но чтобы сотрудники в Odoo могли найти товар ручным вводом "ПТФ Гранта", `template_service.py` должен формировать два пула данных:
1. `name` — Идеальное имя по шаблону.
2. `search_keywords` — Все сырые данные, синонимы, кросс-номера и старые грязные названия. Позже мы будем записывать это в отдельное пользовательское поле (Custom Field) Odoo 19.

**Правило 3: Нормализация метрик (Якоря для Мэтчера)**
Приводите все единицы измерения к строгому стандарту с **обязательным пробелом** перед ними (для правильной токенизации нейросетью). 
*Примеры:* `1000мл` или `1L` → `1 л`. `12V` → `12 В` (или `12 V`). `60Ач` → `60 Ач`.

*(Доп. правило)* **Smart Spaces & Анти-Тавтология:** Если атрибут пустой (Null/None), он бесследно пропускается `(" ".join(filter(None, parts)))`. Двойные пробелы недопустимы. Если Состав ("Акрил") заложен в Тип ("Акриловая мастика"), дубль удаляется программно.
