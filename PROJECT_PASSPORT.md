# Паспорт проекта: NameForge

Краткая справка для людей и ИИ: что это за система, с чем она связана и что нельзя ломать.

---

| Поле | Значение |
|------|----------|
| **Название** | NameForge |

| **Репозиторий / каталог** | `C:\PyProject\NameForge` |

---

## 2. Назначение и бизнес-цель

**NameForge** — PIM-инструмент для: человекочитаемых названий товаров из атомарных характеристик, работы по **матрицам шаблонов** (`NAMING_TEMPLATES.md`) и **синхронизации с Odoo** (чтение карточек и справочников, обновление имён и атрибутов).

Ценность: ускорить массовое обогащение каталога и согласование имён без ручного копипаста в Odoo.

## 4. Системный контекст (логика потоков)

Текущая целевая схема (без МойСклада):

1. **Чтение** из Odoo (XML-RPC): товары, атрибуты, категории и связанные справочники.
2. **Локальный кэш** в БД приложения (SQLite сейчас; на VPS — PostgreSQL через `DATABASE_URL`) для отзывчивого UI.
3. **Редактирование** в интерфейсе: шаблоны, сопоставления, генерация названий.
4. **Запись** обратно в Odoo (XML-RPC): обновление `product.template`, атрибутов и т.п. по правилам проекта.

---

## 5. Технологический стек

### Целевой (NameForge 2.0)

- **Backend:** FastAPI, **SQLAlchemy 2.0 (sync)**, Pydantic v2, `pydantic-settings`
- **Frontend:** React, TypeScript, Vite, Tailwind, **shadcn/ui**, React Hook Form, Zod, TanStack Query
- **БД приложения:** SQLite (`data/autoname.db`) локально → PostgreSQL на проде (тот же код, другой `DATABASE_URL`)
- **Интеграция с Odoo:** XML-RPC (клиент в `backend/app/services/` — по плану в `DEVELOPMENT_PLAN_V2.md`)

### Наследие (не удалять без явного решения)

- **UI:** Streamlit (`app.py`, `pages/`)
- **БД:** прямой SQLite в `src/db.py`
- **МойСклад:** `src/moysklad_client.py`, `src/product_workflow.py` — **исторический референс**, в 2.0 не переносится как основной канал

## 7. Конфигурация (.env)

Минимальный задел для 2.0 (имена могут совпадать с вашим фактическим `.env`):

- `DATABASE_URL` — подключение к БД приложения (SQLite / PostgreSQL)
- `ODOO_URL`, `ODOO_DB`, `ODOO_USER`, `ODOO_PASSWORD` — доступ к Odoo 19 по XML-RPC

Секреты в репозиторий не коммитить.

---

## 8. Что нельзя ломать без осознанной миграции

- `src/`, `app.py` — рабочий legacy до полного паритета функций в 2.0
- `data/autoname.db` — пользовательские статусы, шаблоны, привязки
- Совместимость путей: код ожидает данные в `data/` относительно корня проекта
